# Docker PostgreSQL maps to 5433 (not 5432) to avoid local PG conflicts

- **Symptom:** Tests fail to connect, OR tests connect to the wrong PostgreSQL when both Docker PG and a local Homebrew/installer PG are running.
- **Cause:** Default PostgreSQL port 5432 is often already bound by a system-level Postgres install. The container can't bind, OR if it does, clients connect to the host install instead of the container.
- **Solution:** `docker-compose.yml` maps Postgres to host port 5433. Use `DATABASE_URL=postgres://...@localhost:5433/...` in `.env` and `.env.local`. Never assume 5432.
- **Discovered in:** Event-Driven Notification Hub, DB migration setup (2026-03-31).
- **Affects:** Local dev only. Production sets its own port via env.
